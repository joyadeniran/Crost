# SKILL: pptx — PowerPoint Presentation

## Purpose
Produce a well-structured PowerPoint presentation as a JSON slide manifest.
Your output will be converted to a real `.pptx` file by Crost's artifact transformer.
Do not produce raw XML, binary data, or Python code — output valid JSON only.

## When this skill applies
- Task action contains: `create_presentation`, `create_pptx`, `build_slides`, `generate_slides`
- Task params include `output_format: "pptx"` or `output_format: "presentation"`
- The task label or expected deliverable references a slideshow, deck, or presentation

## JSON output contract

Your entire response MUST be the following JSON structure — no prose before or after:

```json
{
  "skill": "pptx",
  "title": "<Presentation title>",
  "theme": {
    "primary_color": "#<hex>",
    "secondary_color": "#<hex>",
    "font_heading": "<font name, e.g. Calibri>",
    "font_body": "<font name>"
  },
  "slides": [
    {
      "slide_number": 1,
      "layout": "title",
      "title": "<Slide heading>",
      "subtitle": "<Optional subtitle or tagline>",
      "notes": "<Speaker notes for this slide>"
    },
    {
      "slide_number": 2,
      "layout": "content",
      "title": "<Slide heading>",
      "bullets": [
        "<Bullet point 1>",
        "<Bullet point 2>"
      ],
      "notes": "<Speaker notes>"
    },
    {
      "slide_number": 99,
      "layout": "citations",
      "title": "Sources & References",
      "bullets": [
        "<Memo entry reference>",
        "<KB file reference>",
        "<Tool call reference>"
      ],
      "notes": "Auto-generated by Crost. Do not delete."
    }
  ],
  "sources": {
    "memo_ids": [],
    "kb_file_ids": [],
    "tool_calls": []
  }
}
```

## Slide layout types

| Layout | When to use |
|--------|-------------|
| `title` | First slide only — big title + subtitle |
| `content` | Standard bullet-point slide |
| `two_column` | Comparison or split content — add `"left": [], "right": []` arrays |
| `image_placeholder` | When a chart or image is needed — add `"image_description": "<desc>"` |
| `quote` | Single impactful statement — add `"quote": "<text>", "attribution": "<source>"` |
| `divider` | Section break — title only, no bullets |
| `citations` | MANDATORY final slide — lists all sources |

## Structure rules

1. **Slide count**: 8–16 slides for a standard deck. Never fewer than 5, never more than 25.
2. **Bullets per slide**: Maximum 5. If you have more to say, split into two slides.
3. **Bullet length**: Maximum 12 words each. Slides are not paragraphs.
4. **Every slide** must have a `title`. Never leave it empty or null.
5. **First slide** must use `layout: "title"`.
6. **Last slide** must use `layout: "citations"` — always included, even if sources arrays are empty.
7. **Speaker notes** should be 1–3 sentences expanding on the slide content for the presenter.
8. **Theme colors**: choose colors that contrast well. Default to `"#2D4A7A"` (professional navy) and `"#F5A623"` (accent amber) if the founder has not specified brand colors.

## Anti-patterns — never do these

- ❌ Long paragraphs in bullet arrays
- ❌ More than 5 bullets per slide
- ❌ Slides with no title
- ❌ Omitting the citations slide
- ❌ Producing raw XML, base64, or binary content
- ❌ Placeholder text like "Insert content here" — always provide real, useful content
- ❌ Identical bullet points repeated across slides
- ❌ "Thank you" as the final slide — citations slide is always last

## Citation instruction

When you use information from company memos, KB files, or tool results:
1. Record the source in the `sources` object at the root of your JSON.
2. **Always** include a final `citations` layout slide listing those sources in plain language.
3. Format memo references as: `"Company Memo: <title> (<date if known>)"`
4. Format KB file references as: `"Knowledge Base: <file title>"`
5. Format tool call references as: `"Tool: <service>.<action> — <brief description of what was retrieved>"`
